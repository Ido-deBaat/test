Introductie

Wat is R?

R is een programmeertaal speciaal ontwikkeld voor data analyse, mathematiek, statistiek en machine learning met miljoenen gebruikers wereldwijd. Daarnaast is R open source en gratis te gebruiken voor iedereen - altijd. Dit maakt dat R is uitgegroeid tot een uitgebreid ecosysteem met 1000’en pakketten, boeken, tutorials, forum bijdragen en meer resources die jou helpen je alledaagse dataprojecten makkelijker te maken.

Hoe werkt R?

Om te begrijpen hoe R werkt, kun je R het beste vergelijken met een enorme toolkit vol met gereedschappen, apparaten en machines die je kunt gebruiken om je eigen fabriek te bouwen. In de R fabriek zijn de functies de gereedschappen en machines, de data is de grondstof. Wanneer je een beetje tijd investeert in het begrijpen van de gereedschappen en de grondstoffen kun je een fabriek bouwen voor ieder datatype en ieder product.

in dit document leer je hoe je vertrouwd kunt raken met de functies en leer je te begrijpen hoe R data interpreteert.

Assignment

Laten we beginnen met assignment. Dit is uiteindelijk één van de belangrijkste onderdelen van een programmeertaal.

Assignment is niets anders dan het aanmaken van een variabele. Je hebt hiervoor drie zaken nodig: een naam voor je variabele; een assignment operator om R te vertellen dat je een variabele aan het aanmaken bent en een waarde die je aan de variabele wilt toe kennen.

Het ziet er zo uit:

x <- 10

Als je dit zou moeten uitspreken zou je kunnen zeggen “x krijgt 10”. In dit geval is x is de naam voor de variabele, <- is de assignment operator en 10 is de waarde.

Wil je x zien, kun je zeggen print(x) maar dat hoeft niet. Simpelweg x invoeren en runnen is voldoende:

x
## [1] 10

Je kunt x nu dus gebruiken bij alles wat je doet:

x / 2
## [1] 5

Environment

Als je een variabele aanmaakt, beland deze in de zo genaamde global environment. Dit kun je zien als het korte termijn geheugen van R. Alles wat jij aangemaakt hebt staat daar, totdat je R afsluit. Zorg er dus voor dat je de code bewaart waarmee je de variabelen gemaakt hebt in een R(Markdown)-script. Hierdoor kun de variabelen die je nodig hebt voor jouw analyse reproduceren.

volgorde is belangrijk

Het is ook niet onbelangrijk dat R je script altijd van boven naar onder leest. Je moet er dus voor zorgen dat je een variabele pas gebruikt nadat je deze hebt aangemaakt.

Het environment scherm (eerste tab’je links boven in de RStudio IDE) geeft een overzicht van alle variabelen die je aangemaakt hebt tijdens jouw R sessie.

Namen voor variabelen

There are only two hard things in Computer Science: cache invalidation and naming things. -- Phil Karlton

Een variabele hoeft niet persee x, y of z te heten; je mag iedere naam verzinnen, maar er zijn wel regels:

Technische regels

De technische regels hebben simpel weg te maken de werking van R. Houdt je deze regels niet aan, krijg je errors.

  • Een variabele naam mag bestaan uit letters, cijfers, underscores _ en punten ., maar…

  • je moet beginnen met een letter (of een punt, maar variabelen beginnend met een punt zijn verborgen variabelen en daarmee niet zichtbaar in de environment).

  • R is hoofdletter gevoelig! Een kleine x is niet dezelfde variabele als hoofdletter X.

zichtbaar_1 <- 2 # dit is ongeveer de bedoeling
._verborgen <- 1 # dit kan, maar het wordt een verborgen variabele

Ongeschreven regels

Natuurlijk zijn er ook ongeschreven regels voor variabelen namen. Ten eerste is het verstandig om zo veel mogelijk logische en begrijpelijke namen te verzinnen die voor iedereen (inclusief jijzelf over twee jaar) leesbaar zijn. Wordt het te cryptisch en te complex wordt het naar verloop van tijd steeds moeilijker om (je eigen) code te begrijpen:

omzet_q1_2021 is al een stuk beter dan omz21q1, of summary_klantsegment_a vs. ksA.

tip

Wij raden aan zo veel mogelijk snake_case te gebruiken, omdat dit de meest leesbare variant is en het best is opgewassen tegen afkortingen.

Veel variabelen hebben namen met meerdere woorden. In normaal Nederlands plaatsen we spaties tussen de woorden, maar white spaces zijn niet toegestaan. Om de verschillende woorden in één naam toch te kunnen onderscheiden kun je kiezen voor verschillende gevallen van casing:

  • snake_case: alle woorden zijn klein geschreven en worden door underscores gescheiden;

  • period.separated: woorden worden scheiden door punten;

  • camelCase: het eerste woord is klein geschreven, de volgende woorden beginnen met een hoofdletter;

  • PascalCase: ieder woord begint met een hoofdletter.

Assignment operatoren

Er zijn drie operatoren die je kunt gebruiken in R:

  • <-: de meest gebruikte assignment operator binnen R. Om jouw code vergelijkbaar te maken met die van de rest van de R community raden we heel sterk aan deze te ook daadwerkelijk te gebruiken. Het pijltje kun je in RStudio ook vinden onder de shortcut alt - (Linux/Windows) of option - (Mac).

  • -> Het pijltje kan ook de andere kant op wijzen. Mocht je het willen gebruiken moet je wel rekening houden dat de volgorde ook andersom is: waarde -> naam. Toch raden we heel sterk af deze te gebruiken. Deze wordt zelden gebruikt en maakt de code vreselijk verwarrend.

  • =: globaal de meest gebruikte assignment operator. Wordt gebruikt in Python, PHP, java, C, C++, C# etc. Maar gebruik toch maar gewoon het pijltje naar links.

tl;dr

Het kan heel kort: x <- 10. Let op je casing.

Data

Om de R fabriek en alle tools en machines te kunnen gebruiken is het goed om de grondstof eerst goed te kunnen: de data.

In principe is het heel eenvoudig: R werkt met data types die weer samen gevoegd kunnen worden tot data structuren.

  • Data types kun je het beste zien als een type grondstof, zoals hout, metaal, glas.

  • Data structuren zijn objecten zoals een balk, een wiel, een paneel, etc. gemaakt uit één of meerdere van deze grondstoffen.

Datatypes

Laten we beginnen met het kleinste van het kleinste; de data types. Dit is in R vrij eenvoudig, omdat het er maar zes zijn waarvan je er maar drie echt hoeft te begrijpen:

  • logical: de booleaanse waardes TRUE of FALSE;

  • numeric: getallen;

  • character: alle leestekens.

Logical

Het Logical datatype kent maar twee waardes TRUE en FALSE beiden geschreven met hoofdletters. Schrijf je de woorden niet met hoofdletters herkent R het niet als logical value.

Je mag TRUE en FALSE wel afkorten tot T en F.

T
## [1] TRUE

Dit is de basis van het logical datatype, maar er is meer aan de hand:

TRUE + TRUE + FALSE
## [1] 2

Probeer te bedenken waarom dit het geval is…

Numeric

Numeric gaat over cijfers en getallen. Let wel op: R accepteert alleen de punt . als scheidingsteken voor de decimalen! Je kunt dus geen comma’s gebruiken. 1.9999 gaat goed, 1,9999 geeft errors.

Verder is het goed om te weten dat numeric in principe een verzamelbegrip is voor de datatypes integer en double. Daarbij gaat het bij:

  • integer: alleen ronde getallen, zonder decimale getallen en bij

  • double: om getallen met eventuele decimalen getallen.

Het technische verschil is voor nu niet heel belangrijk omdat R standaard overal een double van maakt en daarna alles als numeric weergeeft, maar de termen integer en double worden veel gebruikt in de helpfiles.

met de typeof() functie kun je kijken van welk datatype (of datastructuur) je data gemaakt is:

typeof(1.0001)
## [1] "double"
typeof(1)
## [1] "double"

R is lui.

Character

Character accepteert alle leestekens, mits deze tussen aanhalingstekens geplaatst worden. Een verzameling leestekes (characters) tussen aanhalingstekens noemen we een string.

"hello world... 123... &*$@ etc... dit is een string"
## [1] "hello world... 123... &*$@ etc... dit is een string"

Je kunt in R gebruikmaken van dubbele- of enkele aanhalingstekens. R gebruikt zelf altijd de dubbele.

Dit is pretty simple, tot dat jouw character string aanhalingstekens bevat. In dat geval kun je kiezen voor twee oplossingen:

  • dubbele aanhalingstekens om enkele plaatsen of andersom;

  • aanhalingstekens in de string “escapen”.

Dubbele- en enkele aanhalingstekens afwisselen

Als jouw character string enkele aanhalingstekens bevat, kun je die prima tussen dubbele aanhalingstekens plaatsen:

"tekst met 'enkele' aanhalingstekens, omsloten door dubbele aanhalingstekens"
## [1] "tekst met 'enkele' aanhalingstekens, omsloten door dubbele aanhalingstekens"

of andersom:

'tekst met "dubbele" aanhalingstekens, omsloten door enkele aanhalingstekens'
## [1] "tekst met \"dubbele\" aanhalingstekens, omsloten door enkele aanhalingstekens"

Mocht je je afvragen waarom R black slashes \ voor de dubbele aanhalingstekens in de string geplaatst heeft, lees dan het volgende kopje:

Aanhalingstekens escapen

In computerwereld zijn er heel veel leestekens met een bijzondere functie. De aanhalingstekens zijn daarvan het beste voorbeeld. Deze vertellen in het geval van R: “let op hier komt een character string”. Door een back slash \ voor de aanhalingstekens te plaatsen heffen we de bijzondere functie op. Dit noemen we escaping.

"tekst met \"dubbele\" aanhalingstekens, omsloten door dubbele aanhalingstekens" 
## [1] "tekst met \"dubbele\" aanhalingstekens, omsloten door dubbele aanhalingstekens"

Datatypes veranderen

R kiest het datatype zelfstandig op basis van jouw invoer, maar je kunt datatypes wel veranderen. Zo kun je bijvoorbeeld een character string bestaande uit getallen “coercen” (of wel dwingen) tot numeric. De functies die je hiervoor kunt gebruiken hebben namen zoals as.numeric(), as.character(), as.logical(), etc.

Let wel op dat je je aan de regels van het datatype moet houden. Van "12.5" kun je een numeric maken, van "12,5" niet, dankzij de komma.

as.numeric("12.5")
## [1] 12.5
as.numeric("12,5")
## Warning: NAs introduced by coercion
## [1] NA

R geeft missende waardes aan met NA wat staat voor Not Available. Houdt je je niet aan de regels van de datatypes geeft R een missende waarde terug.

Nog één voorbeeld:

as.logical(1)
## [1] TRUE

Let ook op dat het datatype uitmaakt. 1 + 1 werkt, maar "1" + 1 niet, omdat R numerics niet kan optellen bij characters.

tl;dr

De drie belangrijkste datatypes in R zijn logical, numeric en charater. De voorwaarde die R stelt aan het invoeren van deze datatypes zijn:

  • logical: alleen de waardes TRUE / T en FALSE/ F met hoofdletters;

  • numeric: uitsluitend getallen, met een punt als scheidingsteken voor de decimalen;

  • character: alle leestekens, maar altijd omsloten door aanhalingstekens.

verander je het datatype, let dan goed op deze regels.

Data structuren

Tot nu toe hebben we steeds maar één datapunt gezien, maar aan één datapunt heb je niet veel. Als je wilt werken met meerdere datapunten zul je een constructie nodig hebben om de datapunten samen te kunnen voegen. Een dergelijke constructie noemen we een datastructuur en uiteraard zijn er meerdere datastructuren beschikbaar. De drie belangrijkste zijn:

  • de vector: een één dimensionale opsomming van datapunten;

  • het dataframe: een (SQL achtige) gegevenstabel:

  • de list: een (NoSQL) key-value object.

Daarnaast heb je ook nog de matrix en de array en nog enkele andere specifieke gevallen zoals een datum object, een (machine learning/statistisch) model, een factor en nog veel meer, maar deze zijn niet heel belangrijk op dit moment.

Vectoren

De vector is de simpelste datastructuur van alle. Het is een één dimensionale opsomming van datapunten. Deze opsomming maak je in R met de concatenate (samenvoegen) functie: c().

Tussen de haakjes van c() kun je de waardes plaatsen die je wilt samenvoegen tot een vector, gescheiden door een komma:

Voorbeeld
c("hello", "world")
## [1] "hello" "world"

Let echter wel op: een vector kan maar één data type hebben:

c(TRUE, 10, "a", "b")
## [1] "TRUE" "10"   "a"    "b"

In de output zijn alle datapunten omsloten door aanhalingstekens, oftewel; alle waardes zijn nu van het type character.

Dit komt omdat character het meest robuuste datatype is; het kan immers alle leestekens bevatten. Numeric is daarentegen weer robuuster dan logical omdat het alle getallen kan bevatten en logical slechts de getallen 0 en 1.

c(TRUE, 10)
## [1]  1 10

Dataframes

Het dataframe is een gegevenstabel zoals je deze vaker tegen gekomen bent. De gegevenstabel bestaat uit de volgende onderdelen:

  • een koprij met de kolomnamen;

  • één of meerdere kolommen;

  • en één of meerdere rijen.

Verder is het goed om te weten dat een kolom hetzelfde is als een vector. Hierdoor kan een kolom dus ook maar één datatype hebben.

Je maakt dataframes met read functies zoals read.csv(), met de data.frame() functie of enkele equivalenten daarvan zoals tibble() van het {dplyr} pakket.

Voorbeeld

De data.frame() functie werkt in de basis hetzelfde als de concatenate functie; je kunt een opsomming maken van vectoren en zo kolommen aan elkaar plakken:

fruit_naam <- c("appels", "peren", "bananen")
fruit_prijs <- c(2.99, 3.45, 1.90)

data.frame(fruit_naam, fruit_prijs)
ABCDEFGHIJ0123456789
fruit_naam
<fct>
fruit_prijs
<dbl>
appels2.99
peren3.45
bananen1.90
3 rows

De variabele namen van de vectoren fruit_naam en fruit_prijs worden gebruikt als kolomnamen, de vectoren zelf vormen de kolommen.

Als je niet de variabele namen wilt gebruiken, maar liever andere namen opgeeft kun je ook gebruikmaken van de zogenaamde tag = value syntax. Daarbij is de tag de nieuwe naam van de kolom en de value de naam van de vector die je wilt gebruiken:

data.frame(naam = fruit_naam, prijs = fruit_prijs)

data.frame(naam = fruit_naam, 
           prijs = fruit_prijs, 
           bio = c(FALSE, TRUE, FALSE))

Lists

Niet alle data komt altijd in de vorm van een tabel, soms heeft je data meer de vorm van een opsommingslijst, bijvoorbeeld voor een bestelling in een pizza restaurant:

  • pizza:

    • pizza_naam: hawaii,

    • prijs: 12.95

    • ingrediënten:

      • ham
      • kaas
      • ananas

  • pizza:

    • pizza naam: tonno

    • prijs: 13.49

    • ingrediënten:

      • mozzerella
      • tonijn
      • ui
      • kappertjes

  • etc…

Deze data bestaat uit (genestelde) key value paren. Dit is typisch voor NoSQL data. Deze data kan feitelijk uit alles bestaan: key value paren van verschillende data types, opsommingen (zoals vectoren), hele tabellen en andere genestelde objecten. De list is de datastructuur binnen R voor dit type data. Je kunt werkelijk alles in een list stoppen.

Voorbeeld

Je kunt lists maken met de list() functie. De keys en values voer in je met de tag = value syntax, die ook beschikbaar is in de data.frame() functie. tag = value is uiteindelijk slechts een andere benaming voor key = value.

ingr_hawaii <- c("ham", "kaas", "ananas")
ingr_tonno <- c("mozzerella", "tonijn", "ui", "kappertjes")

list(
  pizza = list(
    naam = "hawaii",
    prijs = 12.95,
    ingredienten = ingr_hawaii
  ),
  pizza = list(
    naam = "tonno",
    prijs = 13.49,
    ingrdienten = ingr_tonno
  )
)
## $pizza
## $pizza$naam
## [1] "hawaii"
##
## $pizza$prijs
## [1] 12.95
##
## $pizza$ingredienten
## [1] "ham"    "kaas"   "ananas"
##
##
## $pizza
## $pizza$naam
## [1] "tonno"
##
## $pizza$prijs
## [1] 13.49
##
## $pizza$ingrdienten
## [1] "mozzerella" "tonijn"     "ui"         "kappertjes"

tl;dr

Voor ieder type data is er een passend formaat in R:

  • vectoren: simpele opsommingen > c(), seq(), rnorm() en meer;

  • dataframes: (SQL) tabellen > read.csv(), data.frame(), en meer;

  • lists: (NoSQL) key value pairs > list(), read_json(), read_xml, etc.
